# 模块文档

## 用户端模块

用户端入口是 `front-page/app/page.tsx`，核心实现位于 `front-page/src/features/lottery/lottery-app.tsx`。

- 登录与会员：游客登录、积分展示、每日签到、分享奖励、月卡状态、战令信息。
- 系列与抽盒：系列列表、活动横幅、概率公示、单抽、十连、软/硬保底、UP 池结果展示。
- 收藏与交换：用户库存、系列进度、重复款识别、交换挂单、接受交换、合成入口。
- 商业化：积分商店、道具库存、首充礼包、周卡/月卡/季卡、战令购买与领奖接口。
- 社交裂变：邀请链接、好友助力、组队开盒、礼物赠送与收礼列表。
- 拼图与抢购：拼图模板、拼图进度、拼合领奖、拼图队伍、限时抢购。
- 运营活动：活动列表、活动奖励展示、参与和领奖 API。

## 管理端模块

管理端入口是 `front-page/app/admin/page.tsx`，核心实现位于 `front-page/src/features/admin/admin-app.tsx`。

- 管理登录：环境变量管理员账号密码，返回后台 token。
- 总览：用户数、抽奖数、中奖数、活动列表、最近抽奖。
- 活动管理：活动列表、快速创建活动、进入礼品管理。
- 礼品管理：按活动查看礼品、快速创建礼品。
- 概率配置：读取活动，保存默认软/硬保底和 UP 池配置。
- 发奖管理：查看履约任务，审核通过并更新状态。
- 抽奖记录：查看抽奖日志。
- 月卡/战令：查看当前赛季和战令任务。
- 商店管理：查看商品列表和首充礼包。

## 后端模块

后端入口是 `backend-server/app/api/v1/[...path]/route.ts`，通过 catch-all Route Handler 承载 `/api/v1/**`。

- `api-response.ts`：统一响应、CORS、Bearer token 解析。
- `errors.ts`：业务错误和 HTTP 状态映射。
- `lottery-service.ts`：业务服务层，编排鉴权、抽盒、积分、商店、社交、拼图、活动等流程。
- `memory-store.ts`：内存仓储和种子数据，实现用户、会话、活动、奖品、库存、积分、商店、战令、社交、拼图、抢购、运营活动。
- `probability.ts`：Alias Method 抽样、软/硬保底、UP 池状态。
- `types.ts`：前后端 API 契约的后端源类型。

## 当前边界

本次对齐遵循“不修改现有架构”：仍然使用 Next.js Route Handler、React Query 和内存态 `MemoryStore`。数据库表结构作为逻辑设计文档输出，后续生产化再迁移到持久化存储。
