## 概述

本次更新围绕**用户管理体系**继续补齐登录系统之后的账号闭环：后端扩展了用户状态、手机号验证、账户资料、登录/状态日志和后台用户管理接口；前端管理台新增用户管理页；用户端开始展示手机号与账号状态提示；同时将原「用户管理设计」拆分为用户端、后台端和第三方认证接入三份文档。

这次改动把“手机号作为平台账号、微信作为第三方身份绑定、账号状态约束资产行为”的设计落到了 MVP 代码中，为后续 MySQL/Redis、真实短信服务商、运营商一键登录和更完整的风控审计做准备。

## 一、新增功能

### 1.1 用户账号模型扩展

**后端类型扩展**（`backend-server/src/server/types.ts`）：

| 能力 | 新增/扩展类型 |
|------|---------------|
| 用户状态 | `UserStatus = pending_phone / active / frozen / disabled / cancelled` |
| 注册来源 | `RegisterSource = wechat / mobile / guest / admin_import` |
| 登录方式 | `LoginType = wechat / mobile_code / phone_login / guest / admin` |
| 用户资料 | `UserProfile`, `UserProfileMutation` |
| 用户账户 | `UserAccount`，包含积分、现金余额、冻结余额和状态 |
| 后台用户管理 | `AdminUserListItem`, `AdminUserListResult`, `AdminUserDetail` |
| 审计日志 | `UserLoginLog`, `UserStatusLog` |
| 手机验证码 | `PhoneCodeRequest`, `PhoneCodeResult`, `PhoneVerifyRequest` |

**存储层扩展**（`backend-server/src/server/memory-store.ts`）：

- 新增 `userProfiles`、`wallets`、`loginLogs`、`statusLogs`、`phoneCodes` 等内存结构。
- 用户创建时写入默认资料、积分账户、现金钱包和登录日志。
- 用户字段从单一 `phone` 兼容扩展为 `mobile` + `phone`，并记录 `status`、`register_source`、`mobile_verified_at`、`last_login_at`、`updated_at`。
- 微信未绑定手机号时创建 `pending_phone` 用户，绑定手机号后激活为 `active`。
- `disabled` / `cancelled` 用户会撤销未过期 session，`cancelled` 会脱敏昵称、头像和手机号。

### 1.2 手机验证码登录/绑定预留

**新增接口**：

| 功能 | 方法 | 路径 |
|------|------|------|
| 发送手机验证码 | `POST` | `/api/v1/auth/phone/code` |
| 校验手机验证码 | `POST` | `/api/v1/auth/phone/verify` |

**开发模式行为**：

- 验证码有效期为 5 分钟。
- `SMS_PROVIDER=mock` 或未配置短信密钥时，返回开发验证码 `123456`。
- 校验成功后，已存在手机号直接登录；不存在手机号则复用当前手机号用户创建逻辑。
- 保留已有 `POST /api/v1/auth/phone-login`，用于演示和兼容。

**相关错误码**：

| 错误码 | 含义 |
|--------|------|
| `phone_code_invalid` | 验证码错误或已过期 |
| `phone_already_bound` | 手机号已绑定其他账号 |

### 1.3 微信登录手机号绑定增强

**调整点**：

- `wechatLogin(code)` 对新微信用户不再默认视为完整账号，而是创建 `pending_phone` 用户。
- 登录结果返回 `need_phone`，用于前端引导手机号验证。
- 微信 `session_key` 继续保存，用于后续解密手机号。
- `wechatBindPhone(openid, encryptedData, iv)` 解密手机号后会：
  - 校验手机号是否被其他用户绑定。
  - 将当前用户激活为 `active`。
  - 写入 `mobile_verified_at`。
  - 记录 `pending_phone -> active` 状态日志。

### 1.4 用户资料与账户接口

**新增/增强接口**：

| 功能 | 方法 | 路径 |
|------|------|------|
| 当前用户账户 | `GET` | `/api/v1/me/account` |
| 更新用户资料 | `PATCH` | `/api/v1/me/profile` |
| 用户退出登录 | `POST` | `/api/v1/auth/logout` |

`GET /api/v1/me/account` 返回用户基础信息、资料、会员积分、现金余额、冻结余额和账号状态。用户端已经接入该接口，用于展示昵称、脱敏手机号和账号状态提示。

### 1.5 后台用户管理接口

**新增后台 API**：

| 功能 | 方法 | 路径 |
|------|------|------|
| 用户列表 | `GET` | `/api/v1/admin/users` |
| 用户详情 | `GET` | `/api/v1/admin/users/:id` |

| 用户登录日志 | `GET` | `/api/v1/admin/users/:id/login-logs` |
| 用户状态日志 | `GET` | `/api/v1/admin/users/:id/status-logs` |

**当前能力**：

- 后台积分调整会检查余额不能为负，并写入 `admin_adjust` 积分流水。
- 禁用或注销用户时会撤销未过期 session。

## 三、服务层与资产限制

### 3.1 账号状态拦截资产行为

`LotteryService` 新增 `assetUserFromToken()`，资产类操作不再只校验 token，还会调用 `MemoryStore.assertAssetAllowed()`。

当前被拦截的状态：

| 状态 | 行为 |
|------|------|
| `pending_phone` | 禁止抽盒、兑换、购买、赠礼、领奖等资产操作 |
| `frozen` | 允许登录查看，但禁止资产变动 |
| `disabled` | 禁止登录或禁止继续操作 |
| `cancelled` | 禁止登录或禁止继续操作 |

**涉及的资产类能力**包括抽盒、交换、兑换、签到、分享奖励、月卡/战令/商店购买、礼物赠送/接收、拼图合成、限时购、活动领奖等。

新增错误码：

| 错误码 | 含义 |
|--------|------|
| `user_status_forbidden` | 当前账号状态不允许执行该操作 |
