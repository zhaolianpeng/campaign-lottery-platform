# 更新日志 — 支付模块封装与后端接入

> **日期**：2026-05-27  
> **标签**：`payment-module-v1`  
> **范围**：独立支付包、后端 API 网关、接入文档（未改动现有聚合路由与 C 端抽奖主流程）

---

## 概述

本次更新新增独立 npm 包 `payment-module`，封装微信支付与支付宝的统一下单、回调验签、查单与退款能力，并根据访问端（手机 / 电脑）自动选择支付呈现方式：手机端拉起本地支付 App（H5 / WAP / 微信 JSAPI），电脑端返回扫码内容。

后端通过**新增** `app/api/v1/payments/**` 路由对接该模块；前端仍只调用后端 HTTP 接口，不直接依赖 `payment-module`。默认 `PAYMENT_ENABLED=false`，不启用时不影响现有业务接口。

## 一、新增：独立支付模块 `payment-module/`

### 1.1 包结构与能力

| 路径 | 说明 |
|------|------|
| `payment-module/package.json` | 独立包 `@campaign-lottery/payment-module` |
| `payment-module/src/index.ts` | 对外导出 |
| `payment-module/src/core/payment-service.ts` | `createPaymentModule` 主入口 |
| `payment-module/src/core/client-platform.ts` | User-Agent 识别与呈现方式选择 |
| `payment-module/src/channels/wechat-pay-adapter.ts` | 微信支付 V3（Native / H5 / JSAPI） |
| `payment-module/src/channels/alipay-adapter.ts` | 支付宝（precreate / WAP） |
| `payment-module/src/store/memory-order-store.ts` | 内置内存订单与状态机 |
| `payment-module/config/payment.config.example.json` | 配置模板（商户号、密钥路径、回调路径） |
| `payment-module/config/payment.config.mock.json` | Mock 联调配置 |
| `payment-module/README.md` | 模块使用说明 |

### 1.2 对外 API（模块层）

| 函数 | 说明 |
|------|------|
| `createPaymentModule({ configPath })` | 加载 JSON 配置并创建实例 |
| `createCheckout(input)` | 创建收银台（自动或强制 `presentation`） |
| `handlePaymentNotify(channel, headers, body)` | 验签并推进订单为 `paid` |
| `queryOrder(orderNo, { syncChannel })` | 查单，可选渠道同步 |
| `requestRefund(input)` | 发起退款 |
| `detectClientPlatform(userAgent)` | `mobile` / `desktop` |
| `detectPlatformForChannel(userAgent, channel)` | 推荐 `qrcode` / `redirect_h5` / `wechat_jsapi` |

### 1.3 呈现方式

| 访问端 | 微信 | 支付宝 |
|--------|------|--------|
| 手机 + 微信内置浏览器 | `wechat_jsapi` | — |
| 手机 + 其他浏览器 | `redirect_h5`（H5） | `redirect_h5`（WAP） |
| 电脑 | `qrcode`（Native） | `qrcode`（precreate） |

### 1.4 配置与安全

- 微信/支付宝凭证、API 密钥、证书路径均通过 `payment.config.json` 配置，不入库。
- `mock: true` 时使用进程内临时 RSA 密钥，无需真实证书即可联调。
- 回调验签：微信 V3 平台证书 + AES-GCM 解密；支付宝 RSA2 参数验签。
- 创建订单支持 `client_request_id` 幂等。

### 1.5 测试与构建

```bash
cd payment-module
npm install
npm test    # 平台识别 + mock 收银台/回调
npm run build
```

## 二、新增：后端支付 API 网关

### 2.1 依赖与构建

**`backend-server/package.json`**：

```json
"@campaign-lottery/payment-module": "file:../payment-module"
```

**`backend-server/next.config.mjs`**：

- 增加 `transpilePackages: ['@campaign-lottery/payment-module']`

### 2.2 服务端封装

| 文件 | 说明 |
|------|------|
| `backend-server/src/server/payment-gateway.ts` | 读取 `PAYMENT_ENABLED` / `PAYMENT_CONFIG_PATH`，单例 `PaymentModule`，错误映射 |
| `backend-server/src/server/payment-http.ts` | Bearer 鉴权、`User-Agent` / 客户端 IP、回调 raw body 处理 |

### 2.3 HTTP 接口（新增路由，未改 `[...path]`）

| 方法 | 路径 | 登录 | 说明 |
|------|------|------|------|
| `GET` | `/api/v1/payments/config/public` | 否 | 支付是否启用、可用渠道 |
| `GET` | `/api/v1/payments/platform` | 否 | 根据 UA 推荐呈现方式 |
| `POST` | `/api/v1/payments/orders` | 是 | 创建收银台 |
| `GET` | `/api/v1/payments/orders/:orderNo` | 是 | 查单（`?sync_channel=true` 可选渠道查单） |
| `POST` | `/api/v1/payments/wechat/notify` | 否 | 微信支付异步通知 |
| `POST` | `/api/v1/payments/alipay/notify` | 否 | 支付宝异步通知 |

路由文件目录：

```text
backend-server/app/api/v1/payments/
├── config/public/route.ts
├── platform/route.ts
├── orders/route.ts
├── orders/[orderNo]/route.ts
├── wechat/notify/route.ts
└── alipay/notify/route.ts
```

### 2.4 环境变量

**`backend-server/.env.example`** 新增：

```env
PAYMENT_ENABLED=false
PAYMENT_CONFIG_PATH=config/payment.config.json
```

**`backend-server/config/payment.config.example.json`**：后端侧配置模板，回调路径与上述 API 对齐。

## 五、启用与联调

### 5.1 启用支付

```bash
# 1. 构建支付模块
cd payment-module && npm run build

# 2. 安装后端依赖
cd backend-server && npm install

# 3. 复制配置
cp config/payment.config.example.json config/payment.config.json
```

在 `backend-server/.env.local` 中：

```env
PAYMENT_ENABLED=true
PAYMENT_CONFIG_PATH=config/payment.config.json
```

本地 mock 可将配置中 `mock` 设为 `true`。

### 5.2 Mock 回调示例

创建订单后，可向微信回调地址 POST：

```json
{
  "out_trade_no": "<order_no>",
  "transaction_id": "wx_mock_1",
  "amount": 990
}
```

再 `GET /api/v1/payments/orders/<order_no>` 确认 `status` 为 `paid`。

## 六、前端接入说明（本次未改前端代码）

前端应通过现有 `apiRequest` 调用后端，例如：

- `GET /api/v1/payments/config/public` — 是否展示支付入口
- `POST /api/v1/payments/orders` — 创建收银台
- 按 `presentation` 渲染二维码、跳转 URL 或调起微信 JSAPI

详见 `docs/payment-module-integration.md`。

## 七、已知限制与后续建议

| 项 | 说明 |
|----|------|
| 订单存储 | 当前为 `payment-module` 内存仓储，进程重启丢失；生产需按 `docs/payment-module-design.md` 落库 MySQL |
