## 二、Bug 修复

### 2.1 [严重] API 请求到手机 localhost 导致 Load failed

**问题**：`src/client/api.ts` 中 `API_BASE_URL` 默认值为 `'http://localhost:18100'`

**影响**：用户在手机上访问页面时，`localhost` 指向**手机自身**，所有 API 请求全部失败，控制台报 "Load failed"

**修复**：改为空字符串 `''`（同源请求），通过 Nginx `/api/v1/` 反向代理到 `:18100`

**文件**：
```
front-page/src/client/api.ts | 2 +-
```

```diff
-const API_BASE_URL = process.env.NEXT_PUBLIC_API_BASE_URL ?? 'http://localhost:18100';
+const API_BASE_URL = process.env.NEXT_PUBLIC_API_BASE_URL ?? '';
```

---

### 2.2 [严重] 白屏 — CSS 文件 404

**问题**：Next.js 页面引用 `/_next/static/chunks/xxx.css`，但 Nginx 将该路径代理到 `localhost:3000`，而 `next start` 对 `/_next/static/` 返回 404

**影响**：所有样式丢失，页面白屏，元素堆积

**修复**：修改 Nginx 配置，`/_next/static/` 改为 `alias` 直接从磁盘提供静态文件

**文件**（服务器 `/etc/nginx/sites-available/gaokao-api`）：

```diff
 location /_next/static/ {
-    proxy_pass http://127.0.0.1:3000;
-    proxy_http_version 1.1;
-    proxy_set_header Host $host;
+    alias /home/ubuntu/campaign-lottery-platform/front-page/.next/static/;
     expires 365d;
     add_header Cache-Control "public, immutable";
 }
```
