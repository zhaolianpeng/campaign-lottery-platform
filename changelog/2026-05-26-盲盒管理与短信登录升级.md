## 概述

本次更新围绕**盲盒管理后台**和**用户端手机号登录**继续补齐平台运营能力：管理端将原“活动/礼品”语义统一为“盲盒/奖品”，新增盲盒发布校验、奖品展示字段、保底配置合法性校验和对应设计文档；用户端则从“本机号码一键登录”的演示交互，升级为基于短信配置的手机号登录表单。

## 一、后端更新

### 1.1 盲盒发布校验接口

**新增接口**：

| 功能 | 方法 | 路径 |
|------|------|------|
| 盲盒发布校验 | `POST` | `/api/v1/admin/campaigns/:id/validate` |

**返回类型**：`CampaignPublishValidation`

| 字段 | 含义 |
|------|------|
| `campaign_id` / `campaign_name` | 被校验的盲盒信息 |
| `prize_count` | 当前盲盒奖品总数 |
| `active_prize_count` | 已上架奖品数 |
| `total_stock` | 已上架奖品库存合计 |
| `total_weight` | 已上架奖品权重合计 |
| `can_publish` | 是否满足发布条件 |
| `errors` | 阻断发布的问题 |
| `warnings` | 不阻断但建议修正的问题 |

**当前校验规则**：

- 盲盒名称和标识不能为空。
- 结束时间必须晚于开始时间。
- 盲盒至少需要配置一个奖品。
- 盲盒至少需要一个已上架奖品。
- 至少存在一个库存大于 0 且权重大于 0 的可抽奖品。
- 可抽奖品权重合计必须大于 0。
- 缺少 Banner、简介或上架奖品图片时给出发布警告。
- 如果配置了保底，会同步校验保底参数合法性。

### 1.2 奖品字段扩展

`PrizeMutation` 新增管理端可写字段：

| 字段 | 用途 |
|------|------|
| `image_url` | 奖品图片，用于用户端展示和发布校验 |
| `sort_order` | 奖品排序，用于后台维护奖品池顺序 |
| `display_prob` | 展示概率，用于后续概率公示 |

`MemoryStore.createPrize()` 已写入以上字段，并且在创建和查询奖品前会确认所属盲盒存在，避免孤儿奖品数据。

---

### 1.3 保底配置合法性校验

`MemoryStore.updatePityConfig()` 在保存前新增校验：

- 硬保底次数必须大于等于软保底起始次数。
- `target_prize` 必须属于当前盲盒。
- `up_prize_id` 必须属于当前盲盒。

校验失败时会抛出 `invalid_pity_config`，避免后台保存跨盲盒奖品或明显不合法的概率配置。

---

### 1.4 手机号登录策略收紧

`GET /api/v1/config/public` 新增短信配置公开信息：

| 字段 | 含义 |
|------|------|
| `sms.provider` | 当前短信服务商配置 |
| `sms.mock_enabled` | 是否处于 mock 短信模式 |

`POST /api/v1/auth/phone-login` 现在仅允许在 `SMS_PROVIDER=mock` 时使用。非 mock 配置下会返回：

| 错误码 | 含义 |
|--------|------|
| `phone_verification_required` | 请使用短信验证码登录 |

这使开发模式仍可快速登录，生产模式则引导用户走 `POST /api/v1/auth/phone/code` + `POST /api/v1/auth/phone/verify` 的验证码流程。
