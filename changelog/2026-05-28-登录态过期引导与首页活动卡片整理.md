> **范围**：优化 C 端登录态过期后的回流路径，并整理首页“系列”页活动卡片展示与登录入口体验

## 概述

本次更新聚焦两个高频体验问题：

1. 用户登录态过期后，原页面会出现接口 401、内容空白或直接展示 `unauthorized`，缺少明确回到登录的引导。
2. 首页“系列”页顶部活动卡片长期使用更重的运营样式，和下方系列卡片尺寸、内边距不一致，移动端容易出现视觉噪音和信息密度不均。

本次改造后，前端会在鉴权失效时统一触发“登录已过期”处理，自动回到登录页并展开登录入口；同时顶部活动卡被收敛为与普通系列卡一致的卡片规格，只保留淡背景图层，首页信息层级更稳定。

---

## 一、登录态过期统一处理

### 1.1 API 层统一派发鉴权失效事件

变更文件：

- `front-page/src/client/api.ts`

调整内容：

- 新增 `AUTH_EXPIRED_EVENT` 事件常量。
- 当带 `token` 的请求返回 `401` 时，前端统一派发登录失效事件，而不是仅在单个页面里各自兜底。
- 对后端返回的原始 `unauthorized` 文案做归一化，统一映射为“登录状态已失效，请重新登录后继续。”，避免英文原始错误直接暴露给用户。

收益：

- 鉴权失效从“局部报错”改为“统一前端行为”。
- 降低拼图、活动、会员等页面各自补 401 处理的重复工作。

### 1.2 登录页自动承接过期用户

变更文件：

- `front-page/src/features/lottery/lottery-app.tsx`

调整内容：

- 监听 `AUTH_EXPIRED_EVENT`。
- 触发后自动：
  - 清空失效 `token` / `nickname`
  - 退出试玩态
  - 回到 `series` 首页
  - 展开手机登录区域
  - 展示中文引导提示
- 保留昵称登录表单，同时允许用户继续“先试玩抽盒”。

用户表现：

- 过期后不再停留在空白页或异常页。
- 用户会直接被带回登录界面，并看到清晰的重新登录提示。

## 二、首页活动卡片整理

变更文件：

- `front-page/src/features/lottery/lottery-app.tsx`

调整内容：

- 将“系列”页顶部活动卡收敛为与下方普通系列卡一致的容器规格：
  - 同高度
  - 同 `p-4` 内边距
  - 同圆角、边框、阴影层级
- 去掉额外的重装饰浮层：
  - 右侧统计浮层
  - 顶部强视觉角标
  - 额外 CTA 浮层
- 保留一层低透明度背景图，让活动卡仍具备轻度运营感，但不再破坏首页信息秩序。
- 活动卡内容压缩为普通卡式结构：
  - 标题
  - 描述
  - 底部活动信息 / 奖励文案

收益：

- 顶部活动卡与系列卡视觉统一。
- 移动端不再出现高卡片、信息挤压、视觉重心偏移的问题。
- 首页“系列”页更接近列表页，而不是“海报 + 列表”混排。
